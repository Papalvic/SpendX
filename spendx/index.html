                    Won't import?

                    Most import failures come down to one thing

                    Bank and wallet apps usually add a few summary rows — account name, account number, total credits — above the real transaction table. Most of the time the app detects and skips these automatically. If it can't (or the file itself is damaged), you can fix it yourself in under a minute by editing the spreadsheet directly.

                    How to edit it

                    Works in Excel, Google Sheets, or the Sheets/Excel app on your phone
                    1

                    Open the file

                    Open the downloaded statement in Excel, Google Sheets, or a spreadsheet app on your phone.

                    2

                    Find the real header row

                    Scroll down until you see the row with column titles like Date, Description, Debit, Credit, or Balance. That's your header row.

                    3

                    Select and delete everything above it

                    Select every row above the header row — account name, account number, opening balance, totals, anything else — and delete those rows entirely (not just the text). On mobile: tap the first row number, drag to the last row above the header, then delete rows.

                    4

                    Confirm the header is now row 1

                    The column titles row should now be at the very top of the sheet, with no blank or text rows above it.

                    5

                    Save and re-upload

                    Save the file (keep it as .xlsx or .csv), then import it again here.

                    💡

                    If the file still won't import after this, it may be genuinely damaged rather than just having extra rows — try re-downloading or re-exporting the statement from your bank or wallet app for a fresh copy.
